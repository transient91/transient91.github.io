A Survey of Automatic Software Vulnerability Detection, Program Repair, and Defect Prediction Techniques

Posted on 2021-07-14

Automatic Software Vulnerability Detection(软件漏洞自动检测)


Code Similarity-Based Vulnerability Detection(基于代码相似性的漏洞检测)

  又称 clone detection(克隆检测)，其核心思想是相似的程序代码具有相似的软件漏洞。四种常见的克隆类型:

  • Type-1 (T1)：克隆是相同的代码片段，可能包含空白、布局或注释的变化
  • Type-2 (T2)：克隆是允许标识符、文字、类型、空白、布局和注释变化的代码片段
  • Type-3 (T3)：克隆允许额外的修改，如更改、添加或删除语句
  • Type-4 (T4)：克隆还包括语义相同但语法不同的代码片段

Grammar-Based Clone Detection Methods(基于语法的克隆检测方法。)

  源代码的表示方式决定了信息提取的上限，进一步限制了模型设计和相似性度量算法的选择，最终影响检测效果。传统的基于文本的表示方法，处理数据只涉及程序代码中注释和空格的去除，主要是基于文本相似性度量方法来检测文本级克隆。然而，基于树的表示方法通过解释器分析程序代码来处理数据，其度量算法考虑了更多的程序结构信息，从语法层面检测代码克隆。
  目前，基于深度学习的方法通过构建抽象语法树截取关键节点信息，将其转化为特征向量或哈希值，然后选择合适的神经网络模型进行训练，最后利用该模型实现克隆比较。

CCLearner

  • 提出了一种基于句子的深度学习代码克隆检测方法，定义了 8 种标记类型。
  • 首先，使用 ANTLR 解析每个方法并提取 C1–C3，并使用 Eclipse ASTParser 为每个方法创建一个抽象语法树，以进一步提取 C4–C8 令牌。然后，计算每种方法中每种类型标记的频率和方法对之间的相似度，将其输入到分类器进行训练，以检测给定代码库的克隆。
  • 由于标记只揭示了语法层面的代码关系，这种方法度量代码函数之间相似性的能力是有限的。
  • 只使用关于源代码的词法信息，不考虑代码结构信息，这使得检测功能克隆对变得困难。

CLDH

  • 提出了一种称为 CDLH 的端到端深度学习框架
  • 以有监督的方式捕获代码片段功能行为的词法和句法信息，并进行哈希，以进一步将实值表示转换为二进制哈希代码。
  • 与 CCLearner 相比，学习到的特征更加紧凑，显著提高了检测效率，节省了存储空间。

White et al.

  • 综合考虑了代码结构和标识符，同时用于建模。
  • 与 CCLearner 基于令牌频率统计的工作不同，本文使用贪婪的方式在多个树结构之间进行转换来表征代码，并使用 RtNN 模型将代码片段中相似的术语映射到相似的连续值向量。此外，该模型基于 RvNN 模型来学习不同粒度级别的代码片段。在实验集中，使用 ANTLR 对源代码进行标记化，使用 RNNLM 工具包 为每个系统训练几个 RtNNs，改变隐藏层的大小和深度。然后，用困惑度作为评价模型质量的代理，模型质量越高，精度越好。

Marastoni et al.

  • 提出了一种新的程序相似性检测方法，将二进制代码转换成图像数据，输入训练模型的 CNN 网络。
  • 同时，作者使用 Tigress C obfuscator (C 混淆器)迭代转换语法对原始数据集进行扩展，对模型进行全面训练。由于 CNN 网络需要查看从二进制文件中提取的整个图像来完成每个文件的分类，二进制可执行文件的大小限制了系统的性能。
  • 在实验阶段，CNN 中的所有权重都是用从标准差设置为 0.1 的正态分布中获取的随机值初始化的，以避免在早期阶段陷入局部极小值，并进一步微调以下参数以提高模型的性能 :test_ratio、 gradient_rate、 limit、random_seed 和 norm。

各种代码表征方法组合

  • 实现了四种代码表示 :标识符、抽象语法树、控制流图和字节码，并为每种表示训练了一个模型。实验结果表明，通过整合不同的模型，可以进一步提高代码克隆检测的准确性。

  与文本和标记方法相比，基于语法的代码表示方法考虑了程序代码的结构信息，对于程序语句的序列转换和微小的修改具有更好的容错性。然而，基于树的方法仍然面临一些缺点。随着程序逻辑结构的增强，构建程序抽象语法树的复杂度增加，进一步导致提取关键代码和向量变换操作的计算开销增加。

Semantic-Based Clone Detection Methods(基于语义的克隆检测方法)

  语义层的代码表示不仅考虑了源代码的语法，还利用了关于程序代码的控制流和数据流信息，解决了代码功能层的代码克隆检测问题。由程序生成的数据流图 DFG、控制流图 CFG 和程序依赖图 PDG 是目前主流的表示方法。
